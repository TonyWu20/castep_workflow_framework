## Implementation Details

### 1. Helper Function: `load_state`

```rust
fn load_state(path: &str) -> anyhow::Result<JsonStateStore> {
    JsonStateStore::load(path)
        .map_err(|_| anyhow::anyhow!("error: state file not found: {}", path))
}
```

Handles missing file error with consistent stderr message format.

### 2. Helper Function: `cmd_status`

Output format:
```
<id>: <STATUS>
...
<id>: Failed (<error>)
...  # for failed tasks
Summary: X completed, Y failed, Z skipped, W pending
```

Implementation:
- Sorts tasks by ID for deterministic output
- Formats failed tasks with error message in parentheses
- Appends summary line using `state.summary()`

### 3. Helper Function: `cmd_inspect`

**With task ID**: Returns detailed view for single task
```
task: <id>
status: <STATUS>
error: <message>  # only if failed
```

**Without task ID**: Lists all tasks (one line per task)

### 4. CLI Integration

Replaced `Commands::Status` and `Commands::Inspect` arms:

```rust
Commands::Status { state_file } => {
    let state = load_state(&state_file)?;
    println!("{}", cmd_status(&state));
    Ok(())
}
Commands::Inspect { state_file, task_id } => {
    let state = load_state(&state_file)?;
    match cmd_inspect(&state, task_id.as_deref()) {
        Ok(out) => { println!("{}", out); Ok(()) }
        Err(e) => { eprintln!("{}", e); std::process::exit(1); }
    }
}
```
